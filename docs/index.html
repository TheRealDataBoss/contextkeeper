contextkeeper

Zero model drift between AI agents. Switch models mid-project. Your AI picks up exactly where you left off.

$ pip install contextkeeper
Python 3.10+ Node 18+ MIT License git required

AI agents forget everything.

Every time you start a new chat, switch models, or hit a context limit, your AI loses all project knowledge — architecture decisions, coding conventions, task progress, what was tried and failed. You re-explain the same things over and over. That's model drift, and it kills productivity.

contextkeeper captures your project's full state vector — decisions, progress, conventions, and active tasks — in a single portable file that any AI agent can bootstrap from instantly.

How it works

01

init

Scaffolds a project context directory with state vector, conventions, and task tracking.

02

sync

Scans your repo and updates the state vector with current project reality — files, deps, structure.

03

bootstrap

Generates a ready-to-paste briefing that brings any AI agent up to speed in one prompt.

04

doctor

Validates your context files for consistency, missing fields, and structural issues.

~/myproject $ contextkeeper init myproject
✓ Created projects/myproject/STATE_VECTOR.json
✓ Created projects/myproject/CONVENTIONS.md
✓ Created projects/myproject/TASKS.md

~/myproject $ contextkeeper sync myproject
→ Scanning repository structure...
→ Detecting dependencies...
✓ State vector updated (14 fields synced)

~/myproject $ contextkeeper bootstrap myproject
✓ Bootstrap prompt copied to clipboard (2.4 KB)

Commands

init

Scaffold a new project context with state vector, conventions, and task files.

contextkeeper init <project-name>
--path --template

sync

Scan your repo and update the state vector with current project state.

contextkeeper sync <project-name>
--deep --dry-run

bootstrap

Generate a briefing prompt to bring any AI agent up to speed instantly.

contextkeeper bootstrap <project-name>
--format --clipboard

doctor

Validate context files for consistency, missing fields, and structural issues.

contextkeeper doctor <project-name>
--fix --verbose

list

List all tracked projects and their current status.

contextkeeper list
--json
